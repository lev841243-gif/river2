### Раздел «Моменты» — главное этого этапа
Сгенерированные картинки убраны, галерея больше не захардкожена: таблица
`GalleryItem`, содержимое правится в **`/admin/gallery`** (загрузка нескольких
файлов, скрыть/показать, порядок стрелками, удаление).

- **Фото приводятся к webp** при загрузке (sharp, ширина до 1600, q80). Снимок с
  телефона — 8–12 МБ, без пережатия он уехал бы посетителям как есть. `rotate()`
  разворачивает по EXIF, иначе портретные снимки лежат на боку; заодно EXIF с
  геометками не попадает на сайт.
- **Видео сжимается в фоне** (`lib/video.ts`, ffmpeg) через `after()` из
  `next/server` — уже после ответа: ffmpeg тратит десятки секунд, столько server
  action не живёт. Файл заменяется **на месте, под тем же именем** — иначе главная
  (ISR, 5 мин) до ревалидации ссылалась бы на исчезнувший файл.
- **Звука на сайте нет вообще**: `-an` вырезает дорожку из файла (экономия
  трафика) плюс `muted` на плеере — страховка для файла, не прошедшего через
  ffmpeg. Для уже сжатых роликов есть `stripAudioInPlace` (`-c:v copy`, без потери
  качества).
- **H.264/mp4 намеренно, не VP9/webm**: mp4 играет везде, включая старые iPhone.
  `+faststart` — иначе браузер не начнёт играть до полной загрузки.
- Пустая галерея скрывает раздел на сайте целиком.
- Итог по весу: 35,4 МБ исходных → 8,6 МБ.

## 3. Грабли этапа — читать перед правками

Здесь то, что выглядит как «лишний код» и провоцирует упростить обратно.

### 1. Загрузки нельзя класть в `public/` — на этом уже обожглись
Next составляет список файлов `public/` **на этапе сборки**. Всё, что заказчик
загрузил после деплоя, отдавалось **404** до следующей пересборки: загрузка не
работала вовсе.

Поймали не сразу — первые файлы я положил до сборки, они показывались, и проверка
ничего не выявила. Нашёл заказчик, загрузив свои.

Поэтому файлы лежат в `uploads/gallery` **вне `public/`** и раздаются
`app/media/gallery/[file]`. Правку Nginx сознательно не делали: она живёт вне git
и потерялась бы при переезде сервера, а обработчик работает одинаково на
localhost и боевом.

В обработчике обязательны две вещи: **Range** (без него браузер не перематывает
видео, а Safari вовсе не играет) и **проверка имени файла** на разделители и `..`
(иначе `..%2F..%2F.env` вычитал бы что угодно с диска).

### 2. Видео с iPhone — HEVC, и без перекодирования оно не показывается
Съёмка с iPhone приходит в **HEVC (h265)**, Chrome и Firefox не декодируют его в
mp4: играет звук, картинки нет. Значит перекодирование держит не только размер,
но и работоспособность.

Отсюда правило в `lib/video.ts`, которое иначе выглядит нелогичным: **если
исходник не H.264, результат берём даже когда он не стал меньше**. Файл поменьше,
который не показывается, хуже файла побольше, который показывается.

Локально (Windows, ffmpeg нет) такие ролики останутся HEVC и не покажутся — это
ожидаемо, а не поломка.

### 3. Grid-аккордеон из FAQ не годится для разделов главной
Приём `grid-rows-[0fr] → [1fr]` требует `overflow-hidden` на содержимом, но он же
обнуляет дорожку `1fr` на сложном контенте (вложенные сетки, `.reveal`) — раздел
открывался в нулевую высоту. Поэтому здесь `max-height` с `ResizeObserver`.
**Не «упрощать» обратно**: выглядит как дублирующий код, но им не является.

### 4. `seed.ts` на боевой затрёт правки из админки
`prisma/seed.ts` делает upsert **всех** полей. На боевой это сбросит цены и ТТХ,
которые заказчик поправил руками. Для боевой есть `scripts/apply-fleet.ts`: он
обновляет только `sortOrder` существующих лодок и создаёт отсутствующие.

### 5. Данные должны попасть в БД ДО сборки
`deploy.sh` генерирует `/` и `/en` статически. Если сначала задеплоить, а потом
поменять флот или галерею, на сайте до 5 минут будет старое. Порядок — в §6.

### 6. Мелочи, которые молча ломали
- **`serverActions.bodySizeLimit`** поднят до 64 МБ: по умолчанию **1 МБ**, и
  видео с телефона не проходило бы без внятной ошибки. Держать в согласии с
  `MAX_FILE_BYTES` в `lib/gallery.ts`.
- **`sharp` закреплён в зависимостях явно** — приходил транзитивно от Next и мог
  пропасть при переустановке, уронив загрузку.

## 6. Как продолжать

- Локальная БД `river2_dev`, тестовый админ `test@dno.local`, тестовые заявки есть.
- `npm run dev` → localhost:3000. `npm run build` проверяет типы.

  сжимается и HEVC не показывается. На боевом ffmpeg есть.

```bash
ssh root@85.198.68.244 'cd /root/dno && git fetch origin && git reset --hard origin/main \
  && npx prisma generate && npx prisma migrate deploy \
  && npx tsx scripts/apply-fleet.ts'      # и/или import-gallery / compress-videos
ssh root@85.198.68.244 'bash /root/dno/scripts/deploy.sh'
```

Если менялся только код — достаточно `deploy.sh` (он сам делает pull, сборку,
подмену, проверку 200 и откат при неудаче).

**Выкатка на боевой — только с одобрения заказчика.**
