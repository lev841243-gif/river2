## 1. Что сделано в этом этапе

### Админка `/admin` (ТЗ, п. 6.3)
Вход, заявки, лодки, клиенты, сводка. Заказчик правит сайт сам, не трогая git.

- **Вход свой**, на `node:crypto` (`lib/auth.ts`): `scrypt` для пароля,
  подписанная HMAC httpOnly-кука для сессии. Не Supabase Auth (его в проекте
  нет) и не Auth.js (beta под Next 16). Причина та же, что у `lib/google-sheets.ts`:
  свой код вместо чужого релиз-цикла.
- **`proxy.ts` — НЕ охрана.** Он лишь проверяет наличие куки для редиректа на
  логин (edge-рантайм, там нет `node:crypto`). Настоящая защита — `requireAdmin()`
  в каждой странице и **каждом server action**.
- **Админы — в таблице `Admin`**, заводятся из консоли: `npx tsx scripts/admin.ts
  add <email> <имя>` (пароль спрашивается скрытым вводом; ещё `passwd`, `off`,
  `rm`, `list`). Открытой регистрации нет намеренно.
- **Оркестрация вынесена в `lib/booking-workflow.ts`**: смена статуса →
  перерисовка карточки в Telegram → зеркало в Google Sheets. Бот и админка ходят
  одним кодом, чтобы не разъехались.
- **Конкуренты за слот** показываются в карточке заявки — следствие правила
  «слот держит только CONFIRMED».
- **ISR включён** (`revalidate = 300` на `/` и `/en` + `revalidatePath` при
  сохранении лодки) — закрыт долг ТЗ п. 7. Без него правка цены не появлялась бы
  на сайте до деплоя.
- Загрузки фото нет (решение заказчика): фото остаются в `public/boats/`.

### Telegram-бот — переведён на кнопки
Диалог `/bron` и перенос времени теперь идут кнопками: катер → день → час →
минуты → длительность → гости. Руками набирают только имя и телефон.

Всё это чинило **один класс ошибки**: действие молча не происходит, человек не
понимает почему. Конкретные случаи, все пойманы на боевом:
- разбор времени требовал только `18:00`, на `18.00` молча вставал → разбор стал
  терпимым (точка/дефис/пробел, минуты необязательны);
- кнопка в устаревшем диалоге роняла `botDraft.update()` с P2025 в лог → теперь
  «диалог устарел, начните заново»;
- ошибка ввода телефона уходила без `force_reply` → следующее сообщение до бота
  не доходило → добавлен `force_reply` на все переспросы;
- панель под полем ввода: подпись кнопки reply-панели при включённой приватности
  до бота не доходит → бот сам выбирает подписи через `getMe`
  (`can_read_all_group_messages`).

### Запуск на основном домене
`prokatkaterov.ru` переключён с прежнего сайта (`45.130.41.20`) на наш сервер
(`85.198.68.244`) сменой A-записей `@` и `www`. Детали и порядок отката —
`DEPLOY_BEGET.md` п. 11. Ключевое:
- один сертификат на три имени (основной, `www`, `new`), автопродление проверено;
- `www` и `new` → 301 на основной домен;
- `/.well-known/acme-challenge/` исключён из редиректа, иначе продление
  сломалось бы через 60 дней;
- вебхук бота переведён на боевой домен;
- старый сайт не тронут, жив как страховка для отката.

### Безопасная выкатка
`scripts/deploy.sh` — одна команда с авто-откатом:
`ssh root@85.198.68.244 'bash /root/dno/scripts/deploy.sh'`.
- Сборка идёт в отдельную папку и подменяется одним `mv` (раньше `rm -rf .next`
  ломал работающий сайт на время сборки).
- `next.config.mjs`: `ignoreBuildErrors` **false** (было true от скаффолда) —
  сборка теперь падает на ошибках типов.
- В конце проверяет 200 и сам откатывается на прошлую сборку, если нет.

## 3. Грабли этапа — короткий список

- **Prisma прячет код ошибки Postgres.** Внутри интерактивной транзакции
  `PrismaClientUnknownRequestError` без поля `code` — `isExclusionViolation()`
  теперь ловит и по тексту. Иначе подтверждение на занятый слот → 500 вместо
  «занято». Било и по боту.
- **React сбрасывает форму** после server action — на логине стирался email при
  опечатке в пароле. Возвращаем email обратно.
- **Одно правило email** для формы входа и скрипта (`adminEmailSchema`): иначе
  скрипт заводил учётку, которой нельзя войти (кириллица в адресе).
- **`tsconfig.json` дописывается сборкой** (include для распознавания типов) —
  `deploy.sh` сбрасывает локальные правки на сервере перед `git pull`, иначе pull
  падает. Та же болезнь, что была с `tsconfig.tsbuildinfo`.
- Полный разбор каждой — в `PROJECT_STATE.md`.
