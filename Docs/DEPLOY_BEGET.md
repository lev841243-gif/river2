# Деплой на Beget — пошаговый runbook

Сайт «Судоходная Компания Дно» (Next.js + PostgreSQL + Prisma). SSH-клиент не нужен — все команды вставляются в **веб-консоль в панели Beget**, остальное делается кликами.

> Если какой-то экран в панели выглядит иначе — пришли скриншот, подскажу точнее под твой интерфейс.

---

## 0. Что уже готово в коде

- Схема БД — `prisma/schema.prisma` (лодки, заявки, статусы, клиенты).
- Миграции — `prisma/migrations/` (init + защита от двойного бронирования).
- Сид всех 15 лодок — `prisma/seed.ts`.
- Бронирование на сайте: календарь занятости, проверка пересечений, API.
- Сайт читает флот из БД, а пока БД нет — из статики (`lib/boats-db.ts`), т.е. не ломается.
- Зависимости и скрипты уже в `package.json` (`postinstall` сам сгенерирует Prisma-клиент).
- Шаблон переменных — `.env.example`.

## ⚠️ Требование к PostgreSQL — проверить ДО деплоя

**Нужен PostgreSQL 13 или выше.**

Защита от двойного бронирования — это exclusion-констрейнт в БД, ему нужно расширение `btree_gist`. Миграция ставит его сама (`CREATE EXTENSION IF NOT EXISTS btree_gist`).

- **PG 13+** — `btree_gist` помечен как *trusted*, ставится владельцем базы **без прав суперюзера**. Проверено: миграции полностью проходят от обычного пользователя-владельца.
- **PG 12 и старше** — `btree_gist` требует суперюзера. `npx prisma migrate deploy` упадёт на второй миграции. Тогда: попросить поддержку Beget выполнить `CREATE EXTENSION btree_gist;` один раз в вашей базе — после этого миграции пройдут.

Проверить версию и доступность расширения (в веб-консоли, подставив свои данные):

```bash
psql "postgresql://ПОЛЬЗОВАТЕЛЬ:ПАРОЛЬ@ХОСТ:5432/ИМЯ_БД" -c "SHOW server_version;"
psql "postgresql://ПОЛЬЗОВАТЕЛЬ:ПАРОЛЬ@ХОСТ:5432/ИМЯ_БД" -c "SELECT name FROM pg_available_extensions WHERE name='btree_gist';"
```

Вторая команда должна вернуть `btree_gist`. Если пусто — расширение не установлено на сервере, нужна поддержка Beget.

Пользователь БД должен быть **владельцем** базы (так и будет, если создавали её через панель).

## 1. Node.js

В панели Beget → раздел **Node.js** (или VPS с преднастроенным Node). Нужна версия **20 или выше**.
Проверка в веб-консоли:

```bash
node -v
```

Если ниже 20 или не установлен — поставить через панель (выбрать версию) или в консоли через nvm (подскажу командой).

---

## 2. База данных PostgreSQL

Панель Beget → **Базы данных → PostgreSQL → Создать**. Запиши 5 значений:
хост, порт (обычно `5432`), имя БД, пользователь, пароль.

Собери из них строку подключения:

```
postgresql://ПОЛЬЗОВАТЕЛЬ:ПАРОЛЬ@ХОСТ:5432/ИМЯ_БД
```

Пришли её мне (или просто вставь в `.env` на шаге 5).

---

## 3. Поддомен (staging)

Панель → **Домены/Поддомены** → создать `new.prokatkaterov.ru`, направить на этот же сервер.
Старый сайт на `prokatkaterov.ru` пока не трогаем — переключим в самом конце.

## 5. Переменные окружения

В папке проекта создать файл `.env` (в git он не попадает). Минимум для старта:

```env
DATABASE_URL="postgresql://ПОЛЬЗОВАТЕЛЬ:ПАРОЛЬ@ХОСТ:5432/ИМЯ_БД"
NEXT_PUBLIC_SITE_URL="https://new.prokatkaterov.ru"
```

Telegram и доступ в админку добавим позже (шаблон — в `.env.example`).

## 6. Установка, миграции, сборка

В веб-консоли, в папке проекта, по порядку:

```bash
npm ci                      # установка зависимостей (+ автогенерация Prisma-клиента)
npx prisma migrate deploy   # создаёт таблицы + ставит btree_gist и констрейнт
npx prisma db seed          # заливает 15 лодок в БД
npm run build               # production-сборка Next.js
```

После `db seed` сайт будет брать флот уже из базы.

**Проверить, что защита от двойного бронирования встала** (без неё два клиента смогут забронировать один слот):

```bash
psql "$DATABASE_URL" -c "SELECT conname FROM pg_constraint WHERE conname='Booking_no_overlap';"
```

Должно вернуть `Booking_no_overlap`. Если пусто — см. раздел про PostgreSQL выше.

---

## 7. Запуск процесса (PM2)

```bash
npm install -g pm2          # один раз, если pm2 ещё нет
pm2 start npm --name dno -- start   # запускает next start (порт 3000)
pm2 save
pm2 startup                 # автозапуск после перезагрузки сервера (выполни команду, которую он подскажет)
```

Проверить, что живёт:

```bash
pm2 status
curl -I http://localhost:3000
```

> Если в панели Beget есть раздел управления Node-приложением — можно вместо PM2 указать команду запуска `npm start` и порт `3000` через интерфейс, тогда перезапуск будет кнопкой.

## 8. Nginx + SSL

- Проксирование поддомена `new.prokatkaterov.ru` → `http://localhost:3000`. В Node-хостинге Beget это обычно настраивается автоматически при регистрации приложения; на VPS — через конфиг Nginx (пришлю готовый блок, если нужно).
- **SSL** — в панели Beget: выпустить бесплатный сертификат Let's Encrypt для поддомена (одна кнопка).

```bash
cd ~/dno
git pull
npm ci
npx prisma migrate deploy   # только если менялась схема БД
npm run build
pm2 reload dno
```

(Сделаю из этого один скрипт `deploy.sh`, чтобы запускать одной командой.)

## 11. Переключение основного домена (в самом конце)

Когда на `new.prokatkaterov.ru` всё протестировано:

1. В панели перенаправить `prokatkaterov.ru` на это же Node-приложение (сменить корень/прокси домена).
2. Выпустить SSL для основного домена.
3. Старый сайт убрать в архив (папку не удалять сразу — пусть полежит).

Ноль простоя: переключение — это смена того, куда смотрит домен, а приложение уже работает.

## Про переменные окружения — важно

`.env` в git не попадает (он в `.gitignore`), поэтому на сервере его нужно создать **руками** — `git pull` его не принесёт. Пароль от боевой БД в репозиторий не коммитить никогда.
